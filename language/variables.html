Variables & Constants

Variables are the fundamental building blocks for storing and manipulating data in Mintas. The language provides flexible keywords and clear rules to ensure your code is both readable and safe.

Declaration Keywords

Mintas offers two ways to declare data: mutable variables and immutable constants.

Mutable Variables (let, so)

Use let or its more expressive alias so to declare variables that can change over time.

let score = 10
so name = "Player1"

score = 25 # Valid reassignment
name = "Player2"

Immutable Constants (const, consta)

Use const or consta for values that should never change. Once assigned, any attempt to modify them will result in a runtime error.

const PI = 3.14159
consta VERSION = "1.0.0"

PI = 4 # Error: ConstantReassignment

Naming Rules (Identifiers)

To keep code consistent and valid, Mintas follows specific naming conventions:

  • Start Character: Must begin with a letter (a-z, A-Z) or an underscore (_).
  • Follow-up Characters: Can include letters, underscores, and numbers (0-9).
  • Case Sensitivity: userValue and uservalue are treated as two different variables.
  • Reserved Words: You cannot use keywords like if, while, or func as variable names.

The 'is' Alias

To make your code read more like natural English, you can replace the assignment operator (=) with the keyword is.

let age is 25
so status is "Active"
const gravity is 9.81

Dynamic Typing

Mintas is dynamically typed, meaning you do not need to declare a variable's type. A variable can hold any kind of data and can even change its data type during its lifecycle (though this is generally discouraged for clarity).

let data = 100      # Currently a Number
data = "Converted" # Now a String
data = empty       # Now Empty

Scope

Variables in Mintas are globally accessible within the script unless they are declared inside a class or as function parameters. Inside a class, properties must be accessed via the this keyword.
